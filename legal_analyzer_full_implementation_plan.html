RAG Pipeline Architecture

This is the brain of your project. Every document upload triggers this exact automated pipeline:

1
User uploads PDF document
React → FastAPI → S3
2
Text is extracted page by page
PyMuPDF
3
Text is chunked (500 tokens, 50 overlap)
LangChain Splitter
4
Embeddings are generated for each chunk
OpenAI Embeddings
5
Vectors are stored in the database
Pinecone / ChromaDB
6
User asks a question via Chat UI
POST /chat/{id}
7
Similarity search retrieves top 5 relevant chunks
Vector Retrieval
8
AI formulates and streams the final answer
GPT-4o

Phase-wise Implementation

1

Foundation + Auth + PDF Upload

Setup project, database, login system, and file upload

Weeks 1–3

Project Setup

Create React app + FastAPI project, folder structure, Git repo, .env file with all secrets.

Database Setup

PostgreSQL on Supabase, create all 6 tables with SQLAlchemy models, run Alembic migrations.

Auth System

Register, login, JWT tokens, protected routes in React, bcrypt password hashing in FastAPI.

PDF Upload & Extraction

React file upload UI, FastAPI multipart endpoint, store file in S3, PyMuPDF text extraction to DB.

  • Install Backend: pip install fastapi uvicorn sqlalchemy alembic pydantic python-jose passlib bcrypt pymupdf boto3 python-multipart
  • Install Frontend: npx create-react-app frontend && npm install axios react-router-dom tailwindcss
  • Environment variables needed: DATABASE_URL, JWT_SECRET, AWS_KEY, AWS_SECRET, S3_BUCKET
  • FastAPI endpoints: POST /auth/register, POST /auth/login, POST /documents/upload, GET /documents
🎯 Deliverable: User can sign up, log in, upload a PDF, and see it listed in their dashboard.
2

RAG Pipeline + AI Analysis Engine

Core AI — embeddings, vector DB, clause extraction, risk scoring

Weeks 4–7

Text Chunking & Embeddings

LangChain splits text into 500-token chunks. Send to OpenAI text-embedding-3-small, store in ChromaDB.

Document Summary

Prompt GPT-4o with first 3000 tokens → return 5-line plain English summary of document type.

Clause Extraction

Structured prompt identifies: penalty, termination, NDA, payment, liability, IP ownership clauses.

Risk Scoring

GPT assigns risk level (low/medium/high) to each clause + overall risk score 1–10 for document.

  • Install AI packages: pip install langchain langchain-openai langchain-community chromadb openai tiktoken
  • Set OPENAI_API_KEY in .env file (create account at platform.openai.com)
  • Build 3 separate LangChain prompt templates: summary.py, clause_extractor.py, risk_scorer.py
  • FastAPI endpoint: POST /analyze/{doc_id} — runs full pipeline, saves to analyses + clauses tables
🎯 Deliverable: Upload any contract → get a summary + color-coded risk clauses + overall risk score out of 10.
3

Chat with Document + Hindi Support + Full UI

Q&A chat, multi-language, polished frontend, PDF viewer side-by-side

Weeks 8–11

Chat Interface & RAG Q&A

React chat UI. Question → retrieve top-5 chunks → feed GPT-4o → stream answer back with source page.

Hindi Language Support

Detect question language using langdetect, add language instruction to prompt, respond in Hindi.

PDF Viewer Integration

Embed react-pdf to show original document alongside analysis, highlight flagged clause pages.

Analysis Result UI

Risk meter component (0–10), clause cards with color (green/yellow/red), summary box, missing list.

  • Install packages: npm install react-pdf and pip install langdetect deep-translator
  • Save every chat message to chat_messages table (role: 'user' or 'assistant') for history
  • React pages to build: /document/:id/analysis and /document/:id/chat — two side-by-side view modes
  • FastAPI endpoint: POST /chat/{doc_id} with body {question, language} — returns streamed answer
🎯 Deliverable: Full working product — upload, analyze, read clauses, ask questions in Hindi or English.
4

PDF Export + Landing Page + Deployment

Report download, public homepage, live deployment, final polish

Weeks 12–14

PDF Report Export

Generate downloadable analysis report with summary, all clauses, risk score using fpdf2 library.

Landing Page creation

Public homepage — hero section, 'how it works' steps, sample demo screenshot, CTA to sign up.

Cloud Deployment

Deploy React to Vercel. Deploy FastAPI to Railway/Render. Use Supabase for DB and Pinecone for vectors.

Testing & Demo Video

Upload real Indian documents, test full flow, record 5-minute demo video for submission.

  • Install PDF package: pip install fpdf2 reportlab
  • FastAPI endpoint: GET /documents/{id}/export — returns downloadable PDF analysis report file
  • Test documents: Delhi rent agreement, IT company offer letter, startup NDA, loan agreement
  • Final GitHub repo should have: README with live URL, setup instructions, and demo video link
🎯 Deliverable: Live public URL, GitHub repo with documentation, 5-min demo video, and project report ready to submit.

Don't skip Phase 1
A solid foundation saves 3x time later. Take a full week on setup even if it feels boring.

Test with real Indian docs
Download a Delhi rent agreement, a standard IT offer letter, and a startup NDA from the internet. Use these as your test documents.
Commit to GitHub daily
Even one small commit per day keeps the project alive and shows activity to recruiters.
